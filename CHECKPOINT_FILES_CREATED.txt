🎯 API ENDPOINTS

GET    /api/code-snapshots/:repoId
       List all snapshots for repository

POST   /api/code-snapshots/create
       Create new checkpoint at specific commit

GET    /api/code-snapshots/:repoId/snapshot/:commitHash
       Get snapshot metadata

GET    /api/code-snapshots/:repoId/state/:commitHash
       Get full repository state (reconstructed from checkpoint)

GET    /api/code-snapshots/:repoId/diff/:fromCommit/:toCommit
       Compare two commits

DELETE /api/code-snapshots/:repoId/prune
       Remove old snapshots

⚡ PERFORMANCE BENEFITS

Without Checkpoints:
  - Analyze commits #900-1000: Replay 1000 commits
  - Memory usage: HIGH (all diffs in memory)
  - Storage: Multiple full copies

With Checkpoints:
  - Analyze commits #900-1000: Replay 100 commits (from checkpoint)
  - Memory usage: LOW (only needed diffs)
  - Storage: 1 snapshot + incremental diffs
  
Improvement: ~10-20x faster, ~5-10x less storage

🔗 KEY CONCEPTS

Checkpoint: Full snapshot of codebase at specific commit
Reconstruction: Recreate state by replaying diffs from checkpoint
Active Checkpoint: Currently used checkpoint per repository
Incremental Snapshots: Checkpoints every N commits
TTL Expiration: Auto-delete after 30 days
